## Эмулятор процессора с гарвардской архитектурой и стековыми инструкциями
Проект реализует программную модель процессора с раздельной памятью команд и данных (гарвардская архитектура) и набором безадресных (стековых) инструкций. 

Архитектура
- Память команд: commandMemory размером 1024 ячейки (UInt)
- Память данных: dataMemory размером 1024 ячейки (UInt)

Стек: располагается в памяти данных по адресам с 500 по 515 (включительно)

Регистры:
- pc – счётчик команд
- sp – указатель стека (смещение относительно начала стека)
- cmp – регистр флагов результата сравнения (биты: 0 – равно, 1 – первое больше, 2 – первое меньше)

| Инструкция | Код (binary) | Описание |
|------------|--------------|----------|
| `PUSH`     | 0001         | Положить число (следующее в памяти команд) на вершину стека |
| `READ`     | 0010         | Прочитать слово из памяти данных по адресу, взятому с вершины стека; результат поместить на стек |
| `WRITE`    | 0011         | Записать значение с вершины-1 по адресу с вершины стека (оба удаляются) |
| `DUP`      | 0100         | Дублировать вершину стека |
| `DROP`     | 0101         | Удалить вершину стека |
| `JMP`      | 0110         | Безусловный переход по адресу, взятому с вершины стека |
| `CMP`      | 1000         | Сравнить два верхних элемента стека, результат в `cmp`, элементы удаляются |
| `INC`      | 1001         | Инкремент вершины стека |
| `DEC`      | 1010         | Декремент вершины стека |
| `ADD`      | 1101         | Сложить два верхних элемента, результат на стек |
| `JE`       | 1011         | Переход, если флаг "равно" установлен |
| `JG`       | 1100         | Переход, если флаг "больше" установлен |
| `JL`       | 1110         | Переход, если флаг "меньше" установлен |
| `OUT`      | 0111         | Вывести вершину стека (сохраняется в список `output`) |
| `HLT`      | 1111         | Останов выполнения |
